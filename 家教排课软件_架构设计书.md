# LifeTrack — 个人时间管理与追踪

> 本项目由「家教排课软件」重构而来，从面向家教机构的多端同步工具，转型为纯本地、跨平台的个人效率管理应用。

---

## 一、项目概述

### 1.1 背景与转型

原项目「家教排课软件」是一款面向家教机构的课程管理 Android App，包含教师/学生管理、排课冲突检测、课消统计、远程同步等功能。考虑到实际使用场景和个人需求，项目进行了大幅精简和转型：

- **去掉服务器依赖**：取消用户登录、远程 API、数据同步，所有数据纯本地存储
- **去掉家教业务概念**：不再区分教师/学生/教室，简化为「课程表 + 任务 + 目标 + 睡眠」四大模块
- **新增跨平台 Web App**：使用 React + TypeScript 重写，支持任何设备的浏览器访问
- **保留 Android 本地版**：作为原生体验的备选，同样纯本地存储

### 1.3 核心功能

| 模块 | 说明 |
|------|------|
| 📊 **总览 Dashboard** | 一屏展示今日日程、重要任务、目标进度、近期睡眠、今日工作量 |
| 📅 **课程表** | 周视图排课，支持拖拽调整位置，支持从任务拖拽创建课程 |
| ✅ **任务追踪** | 待办/进行中/已完成状态，高/中/低优先级，单次/多次排课类型 |
| 🎯 **目标管理** | 设定长期/短期目标，关联任务，自动统计完成进度 |
| 🌙 **睡眠记录** | 入睡/起床时间记录，1-5 星质量评分，平均睡眠统计 |

## 三、Web App 数据模型

### 3.1 数据库（Dexie.js / IndexedDB）

```typescript
// 目标
table goals: {
  id?: number;
  title: string;
  description: string;
  createdAt: string;   // ISO date
  deadline?: string;
  color: string;       // HEX 色值
}

// 任务
table tasks: {
  id?: number;
  goalId?: number;     // 关联目标
  title: string;
  status: 'todo' | 'in_progress' | 'done';
  priority: 1 | 2 | 3; // 低/中/高
  scheduleType: 'single' | 'recurring'; // 单次（安排后隐藏）/ 多次
  createdAt: string;
  completedAt?: string;
}

// 课程/日程
table lessons: {
  id?: number;
  taskId?: number;     // 关联任务（可选）
  title: string;
  dayOfWeek: 0-6;      // 周日=0, 周一=1...
  startHour: 0-23;
  startMinute: 0 | 30;
  durationMinutes: number;
  color: string;
  location?: string;
}

// 睡眠记录
table sleepRecords: {
  id?: number;
  date: string;         // ISO date
  bedTime: string;      // HH:MM
  wakeTime: string;     // HH:MM
  quality: 1-5;
  note?: string;
}
```

### 3.2 核心交互设计

| 场景 | 操作 |
|------|------|
| 任务 → 课程表 | 展开任务面板 → **拖拽**任务到日程格子，或点击「安排」按钮 |
| 调整课程时间 | **拖拽**课程卡片到新的日期/时间段，自动按 30 分钟对齐 |
| 标记完成 | 课程表上点击「✓」按钮，或任务页点击状态圆环 |
| 单次 vs 多次 | 单次任务安排后自动从面板隐藏；多次任务可反复安排 |
| 已完成划掉 | 关联任务已完成的课程自动显示删除线 + 降低透明度 |

## 四、Android App 架构（精简版）

### 4.1 模块依赖图

```
:app
├── :feature:schedule → :core:domain, :design-system, :core-navigation
├── :core:local       → :core:data (实现 DataSource 接口)
├── :core:data        → :core:domain, :core:common
├── :core:domain      → :core:common
├── :design-system     → (仅 Compose 基础库)
└── :core-navigation   → (仅 Navigation 类型)
```

> 已删除的模块：`:core:remote`（Retrofit/网络）、`:feature:student`、`:feature:teacher`、`:feature:settings`

### 4.2 已移除的复杂度

| 原功能 | 状态 | 原因 |
|--------|------|------|
| Retrofit / 远程 API | ❌ 已删除 | 纯本地无需网络 |
| SyncEngine 同步引擎 | ❌ 已删除 | 无服务器 |
| NetworkMonitor | ❌ 已删除 | 无需检测网络 |
| 用户登录/鉴权 | ❌ 已删除 | 纯本地 |
| 学生/教师管理 | ❌ 已删除 | 非个人场景 |
| 课消统计 | ❌ 已删除 | 非家教场景 |
| WebSocket 实时通信 | ❌ 已删除 | 无多端协同 |

## 五、部署与使用

### 5.1 Web App

```bash
cd lifetrack-web
npm install
npm run dev      # 开发服务器 http://localhost:5173
npm run build    # 生产构建 → dist/ 目录
```

**部署**：将 `dist/` 上传到 Vercel / Netlify / GitHub Pages 等静态托管服务。

**PWA**：手机浏览器访问后，选择「添加到主屏幕」，即可像原生 App 一样使用，且支持离线访问。

### 5.2 Android App

```bash
cd tutor-schedule
# 用 Android Studio 打开，Sync Gradle，运行 :app 模块
```
