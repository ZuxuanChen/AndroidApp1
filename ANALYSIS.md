## 产品经理视角 · 问题诊断

### 🔴 P0（核心缺陷）

| 问题 | 影响 | 位置 |
|------|------|------|
| **数据无法导出/备份** | 用户换设备或清除浏览器数据时全部丢失，这是纯本地应用的最大痛点 | 全局缺失 |
| **Pomodoro完成标记逻辑错误** | 番茄钟结束后标记课程完成时，写入的是旧字段 `completedAt`，而系统已迁移到 `completedDates` 做按天追踪。导致Dashboard统计和课程表完成状态不一致 | `PomodoroTimer.tsx:139` |
| **课程时间重叠无检测** | 两个课程可以完美叠在一起，用户看不到 | `ScheduleView.tsx` |
| **空标题/无效数据无校验** | 可以创建标题为空的课程和任务，开始时间可以大于结束时间 | 多个表单 |

### 🟡 P1（体验缺陷）

| 问题 | 影响 |
|------|------|
| `scheduleType` 与 `isRecurring` 字段语义重复 | 代码冗余，维护成本高，容易引发状态不一致 |
| 习惯打卡没有 streak 统计 | 用户看不到连续打卡天数，失去动力 |
| 睡眠统计只有均值，无趋势图 | 数据回顾页缺少睡眠可视化 |
| 无输入防抖 | 快速操作时可能重复提交 |

### 🟢 P2（锦上添花）

| 问题 | 影响 |
|------|------|
| 无系统主题适配 | 深色模式是现代App标配 |
| Android端功能远少于Web端 | 原生体验的优势没有发挥出来 |
| 无提醒/通知机制 | 需要主动打开App才能看到日程 |

## HR视角 · 项目亮点

作为一个资深HR筛简历时，这个项目让我**眼前一亮**的地方：

1. **跨平台全栈能力**：同一个人同时写了 React 19 + TypeScript 的 Web 前端和 Kotlin + Jetpack Compose 的 Android 原生层，且两者架构都很规范
2. **本地优先架构设计**：Dexie.js + IndexedDB（Web）、Room + SQLite（Android）——这不是随便选的，是有明确"不上传服务器"产品决策的技术落地
3. **完整的数据迁移策略**：数据库从 v1 升级到 v8，每次 schema 变化都有 `seedData` 里的迁移逻辑，说明作者考虑过真实用户的数据存续问题
4. **SVG 自绘图表**：StatsView 里自己用 SVG 画柱状图和折线图，不是套个图表库就完事，对坐标计算和可视化有深入理解
5. **MVI 架构落地**：Android 端不是简单的 MVVM，而是完整的状态机（State → Intent → Effect），有单向数据流意识
6. **拖拽交互的完整实现**：HTML5 Drag API + Compose 的 `detectDragGesturesAfterLongPress`，两端的拖拽都有落点计算
7. **成就系统设计**：Badge 系统把用户行为量化成可展示的成就，这是产品经理思维在代码里的体现
8. **模块化的 Gradle 分层**：`core/data`、`core/domain`、`core/local`、`feature/schedule` — 不是把代码堆在一个 module 里

**一句话评价**：这不是一个"跟着教程做"的Demo，而是一个**有产品决策、有架构思考、有真实迭代痕迹**的个人项目。
